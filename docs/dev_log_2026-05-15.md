# Dev Log - 2026-05-15

## Summary

Today shifted development focus toward combat, relic, skill, and visual-feedback iteration. A dedicated Combat Lab was added and made accessible from the run-selection screen, then expanded into the primary testing surface for item combinations, unit forms, status states, trigger feedback, and board-level visual indicators.

## Current Game Progress

- Added a standalone Combat Lab scene with a clean 10x10 board, a controllable test unit, and a stationary zero-health dummy target.
- The Combat Lab can be opened directly from the first "Choose the Run" selection screen, before theme and difficulty selection.
- The lab side panel now supports unit selection, item acquisition, skill/status preview buttons, combat strikes, reset, and item clearing.
- The Combat Lab board now scales to the available screen area instead of using a fixed small tile size.
- The dummy target remains stationary at the top of the board, has 0 HP, never attacks, and displays incoming damage feedback.
- The Combat Lab is now the default surface for near-term combat, item, and skill testing.

## Visual Indicator Work

- Added `docs/pawns_ascent_visual_indicator_system.md` as the visual language reference for statuses, item categories, skills, terrain, rarity, unit states, and event feedback.
- Expanded `BoardView` into a reusable UI-driven indicator renderer.
- Added persistent status icons above units for bleed, burn, shielded, empowered, frozen, shocked, cursed, and weakened.
- Added terrain overlay glyphs for elevated, blessed, fire, cursed, void, and fog terrain.
- Added unit state indicators for ready, low HP, boss, and legacy-style echo states.
- Added one-shot visual helpers for status effects, item triggers, skill effects, evolution, mutation, aura radius, threat lines, skill trails, item rarity, critical hits, and kill feedback.
- Upgraded existing attack and mutation feedback to use the new shape-burst vocabulary.
- Updated tile tooltips to show active statuses, status durations, and unit state labels.

## Current Risks And Gaps

- The new visual indicator layer is still placeholder UI geometry, not final authored sprite work.
- Frozen, shocked, cursed, weakened, and empowered are visible status states, but their full combat rules are not implemented yet.
- Rarity visuals are inferred in the lab from item count; the item data does not yet define actual rarity fields.
- Aura, threat, critical, kill, and rarity effects are available as renderer helpers, but only Combat Lab currently exercises most of them.
- Phase 3 visuals from the indicator document, such as shaders, distortion, ambient effects, and audio-reactive feedback, are intentionally deferred.
- The current visuals rely on ASCII-like glyphs and UI panels; replacing them with authored sprites should be the next visual polish step.

## Recommended Next Work

- Add sprite assets using `docs/pawns_ascent_sprite_requirements.md` as the production checklist.
- Replace status glyphs with small authored status icons.
- Replace shape-burst placeholder glyphs with item-school, skill-type, and event sprites.
- Add actual rarity metadata to item data once item pools expand.
- Implement real mechanics for frozen, shocked, cursed, weakened, and empowered.
- Start expanding Combat Lab into a controlled mechanics workbench with selectable enemy stats, terrain presets, and repeatable damage rolls.
