# Dev Log - 2026-05-06

## Summary

Advanced the playable run loop from a single encounter into a route-driven roguelike flow, tightened the board/HUD layout, improved tactical readability, and changed movement pacing so enemy turns are readable instead of instant AP dumps.

## Changes

- Added a data-driven run path system with floor nodes for combat, elite, relic, rest, story, ambush, and boss encounters.
- Added a path map overlay so the player chooses the next node before combat starts.
- Expanded `GameState` run state for path graph, active node, current floor, and run economy fields.
- Made encounter setup read node/floor rules, including larger boards on deeper floors and elite item support.
- Added responsive layout behavior for side HUD and bottom HUD modes.
- Added tactical HUD details for nearest threat, turn order, relic summaries, tile hover info, movement previews, and strike previews.
- Added board movement animation for player and enemy pieces.
- Changed combat pacing so each unit resolves one meaningful action per phase when enabled by config.
- Added an action lock while turns and movement animations resolve.
- Added immovable terrain support through `blocks_movement` terrain data.
- Added rock and house terrain as blockers with board glyphs and tooltip/tile info messaging.
- Updated movement and spawning so blocked terrain cannot be crossed, landed on, or used as spawn positions.
- Added focused layout and movement test coverage for the new systems.
