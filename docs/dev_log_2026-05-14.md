# Dev Log - 2026-05-14

## Summary

Pawn Ascent has moved from a playable single-run prototype into a fuller roguelike board-combat loop. The current build has route selection, themed run setup, data-driven rewards and events, encounter objectives, legacy scoring, stronger UI responsiveness, and real temporary ally contracts that join combat as AI-controlled units.

## Current Game Progress

- The core board-combat loop is playable: the player selects a path node, enters an encounter, moves and attacks on the board, resolves enemy phases, clears the fight, receives rewards, and advances through the run.
- Runs now use a generated path across seven floors, including combat, elite, ambush, relic, rest, story, shrine, market, anvil, contract, and boss nodes.
- The run setup flow supports mode, theme, and difficulty selection before the path begins.
- Themes currently include Neutral, Thunder, Volcanic, Ocean, Frost, and Void, each with score multipliers, encounter pressure, terrain marks, draft bias, or board mechanics.
- Difficulty affects score and enemy pressure through data-driven enemy stat and count modifiers.
- The board is more expressive: terrain can block movement, mutate over time, reveal fog, create fire/cursed/blessed/elevated pressure, and host optional encounter objectives.
- Encounter objectives can appear on combat nodes and create secondary tactical goals such as caches, seals, or escape squares.
- Economy is now meaningful enough to route around: gold is earned from combat/objectives/events and spent in shops, upgrades, and ally contracts.
- Relics, perks, run stat bonuses, temporary contracts, consequences, and theme effects now feed into the run state instead of being isolated one-off rewards.
- Temporary allies are implemented as real board units. They spawn from contract records, take an Ally phase before enemies, use shared AI/movement/combat systems, keep HP between fights, expire after duration, and disappear if killed.
- The UI has been upgraded for readability: path cards show previews and run context, choices use responsive grids/scrolling, unaffordable options are disabled with cost feedback, combat HUD shows tile/movement/strike info, and allied pieces have distinct board coloring.
- Legacy progression exists: completed runs can record a legacy boss, score tracks best runs by setup, and the end screen shows multiplier context and run history.

## UI And Feel Progress

- The bottom HUD and side HUD scale better across viewport sizes.
- Choice overlays no longer spill off screen as easily and can handle larger option sets.
- Path selection communicates enemy count, reward type, intensity, objective chance, and carried run effects.
- Combat actions have movement animation, attack flash, damage numbers, mutation flash, and float text feedback.
- The player can inspect tiles for terrain, objectives, movement legality, enemy response previews, unit stats, and strike estimates.
- Ally units use friendly chess glyphs and green coloring so they read differently from hostile pieces.

## Current Risks And Gaps

- Ally AI is functional but simple. It shares the enemy AI path and does not yet coordinate with the player or understand objectives.
- Shop and upgrade options are still shallow; they work, but they need more variety and sharper build decisions.
- Theme mechanics exist, but some need stronger identity and more readable feedback during combat.
- Story and world content is still early. The structure supports it, but the event pool needs more authored scenes and consequences.
- Encounter objectives are promising but need more tuning so they feel like tempting route pressure, not random side chores.
- Balance is still rough. Combat simulation passes, but real player pacing, economy values, ally costs, and theme difficulty need hands-on tuning.
- The first-run onboarding is still minimal. The game can start and be played, but it does not yet teach the rules gracefully.

## Recommended Next Work

- Add a tactical intent layer for allies: guard player, pursue weakest enemy, hold objective, or avoid blocking the player.
- Expand market/anvil choices into richer build decisions with rotating stock, rerolls, and stronger theme-specific offers.
- Add a compact run summary panel that shows active theme rules, contracts, omens, objectives, and score multiplier before each node.
- Create more story events and relic descriptions using the existing writing rules, keeping lines short and bleak.
- Add objective-specific UI highlights and clearer post-combat reporting for completed or missed objectives.
- Run a manual balance pass over early floors: gold income, shop costs, ally costs, enemy counts, and objective rewards.
